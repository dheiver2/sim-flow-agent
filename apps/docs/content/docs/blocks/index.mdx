Blocos são os componentes de construção que você conecta para criar workflows de IA. Pense neles como módulos especializados que cada um lida com uma tarefa específica—desde conversar com modelos de IA até fazer chamadas de API ou processar dados.

## Tipos de Blocos Principais

O Mangaba Flow fornece sete tipos de blocos principais que lidam com as funções essenciais dos workflows de IA:

### Blocos de Processamento
- **[Agente](/blocks/agent)** - Converse com modelos de IA (OpenAI, Anthropic, Google, modelos locais)
- **[Função](/blocks/function)** - Execute código JavaScript/TypeScript personalizado
- **[API](/blocks/api)** - Conecte-se a serviços externos via requisições HTTP

### Blocos de Lógica
- **[Condição](/blocks/condition)** - Ramifique caminhos de workflow baseados em expressões booleanas
- **[Roteador](/blocks/router)** - Use IA para rotear inteligentemente requisições para diferentes caminhos
- **[Avaliador](/blocks/evaluator)** - Pontue e avalie qualidade de conteúdo usando IA

### Blocos de Saída
- **[Resposta](/blocks/response)** - Formate e retorne resultados finais do seu workflow

## Como os Blocos Funcionam

Cada bloco tem três componentes principais:

**Entradas**: Dados vindos para o bloco de outros blocos ou entrada do usuário
**Configuração**: Configurações que controlam como o bloco se comporta
**Saídas**: Dados que o bloco produz para outros blocos usarem

<Steps>

## Conectando Blocos

Você cria workflows conectando blocos juntos. A saída de um bloco se torna a entrada de outro:

- **Arraste para conectar**: Arraste de uma porta de saída para uma porta de entrada
- **Múltiplas conexões**: Uma saída pode conectar a múltiplas entradas
- **Caminhos ramificados**: Alguns blocos podem rotear para diferentes caminhos baseados em condições

## Padrões Comuns

### Processamento Sequencial
Conecte blocos em uma cadeia onde cada bloco processa a saída do anterior:
```
Entrada do Usuário → Agente → Função → Resposta
```

### Ramificação Condicional
Use blocos de Condição ou Roteador para criar diferentes caminhos:
```
Entrada do Usuário → Roteador → Agente A (para perguntas)
                              → Agente B (para comandos)
```

### Controle de Qualidade
Use blocos Avaliador para avaliar e filtrar saídas:
```
Agente → Avaliador → Condição → Resposta (se bom)
                               → Agente (tentar novamente se ruim)
```

## Configuração de Blocos

Cada tipo de bloco tem opções de configuração específicas:

**Todos os Blocos**:
- Conexões de entrada/saída
- Comportamento de tratamento de erros
- Configurações de timeout de execução

**Blocos de IA** (Agente, Roteador, Avaliador):
- Seleção de modelo (OpenAI, Anthropic, Google, local)
- Chaves de API e autenticação
- Temperatura e outros parâmetros do modelo
- Prompts de sistema e instruções

**Blocos de Lógica** (Condição, Função):
- Expressões ou código personalizados
- Referências de variáveis
- Configurações de ambiente de execução

**Blocos de Integração** (API, Resposta):
- Configuração de endpoint
- Cabeçalhos e autenticação
- Formatação de requisição/resposta
